X11 on iOS

Introduction

I'm excited to announce the "developer edition" [1] of X11 on iOS. I've successfully compiled and packaged the required dependencies (and some optional packages) for a fully functioning X11 desktop system on iOS 12 and up. All packages (and all their dependencies, as there are very few *nix libraries ported to iOS) have been compiled for arm64 and have been tested on iOS 12.4 and iOS 13.2.

How it works

At the moment, a virtual screen is accessed via a VNC client to an Xvnc[2] instance running on the iDevice. If you're unfamiliar, Xvnc is an X server with a virtual screen that can be accessed via VNC. The best part of this is no drivers are required: input and output are handled by Xvnc.

Why

I want to turn my iPad into a proper C development environment, and a windowing system helps with that. It's a powerful machine with a lot of Unix features, so X11 seemed reasonable. I'd like to eventually run Wayland, but at the moment that seems years off. Being able to compile and run arbitrary Linux applications would be huge -- Firefox, GNOME, Java GUIs: it's all possible in X11[3].

I intend for this project to largely be documentation for building yourself, so in case I disappear and my repo becomes outdated or goes down you can build it yourself.

Current Features

  • X11 on iOS via Xvnc
  • Working window manager (jwm)
  • Loads of other packaged libraries, tools, and applications
  • All available in a Cydia repo

What's next?

  • OpenGL support via Mesa
  • The rest of Xorg applications
  • Proper desktop enviroments like XFCE and GNOME
