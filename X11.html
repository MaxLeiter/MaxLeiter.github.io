X11 on iOS

Introduction

I'm excited to announce the "developer edition" [1] of X11 on iOS. The packages and dependencies for a fully functioning X11 desktop system on iOS 12 and up have been compiled and are avaiable on Cydia. All packages have been compiled for arm64 and have been tested on iOS 12.4 and iOS 13.1. This requires a jailbroken device.

Ths site will serve largely as documentation for building yourself, in case I disappear or lose interest. You can add the Cydia repo below for the deb packages. Please let me know if you run across any issues; it's likely I messed up including a library or something like that.

How it works

At the moment, a virtual screen is accessed via a VNC client to an Xvnc[2]instance running on the iDevice. If you're unfamiliar, Xvnc is an X server with a virtual screen that can be accessed via VNC. The best part of this is no drivers are required: it's all handled by Xvnc.

Why

I want to turn my iPad into a proper development environment, and a windowing system helps with that. It's a powerful machine with a Unix-like OS, so X11 seemed like a reasonable project. X11 allows running arbirtary Linux applications (assuming you can make them compile) — Firefox, IDE's, native file explorers, and more are now possible.

Current Features

  • X11 on iOS via Xvnc
  • Working window manager (jwm)
  • Loads of other packaged libraries, tools, and applications, including meson/ninja and ocaml.
  • All available in a Cydia repo

What's next?

  • OpenGL support via Mesa
  • The rest of Xorg applications
  • Proper desktop enviroments like XFCE and GNOME
  • Experiment with Wayland

Want to contribute?

Interested in using X11 now, or helping make it available sooner? Feel free to email me or find me on IRC as MaxLeiter in #X11iOS on Freenode.
